# AGENTS.md - Directory Policy

The org-wide `.github/AGENTS.md` and `.github/ENGINEERING.md` bind fully here.
This file adds the directory neutrality contract.

## Mission

List Ajar implementations, integrations, tools, bridges, reports, and conformant sites without ranking them.
The directory is factual infrastructure, not marketing copy.

## Directory neutrality

Entries use factual one-liners.
No editorializing.
No marketing adjectives.
No pay-to-play language or promotional claims such as "best", "leading", "simple", "powerful", or "seamless".
Use alphabetical order inside a category unless the README states a category-specific factual order.
Use category order only when categories are explicit and neutral.
All projects use the same listing criteria.

## Entry requirements

Every entry MUST include a working link and state what the project does for Ajar.
Verify entries before merge.
Verification means the link resolves and the project mentions Ajar; if network access is unavailable, state that in the PR evidence.
Dead links are corrected or removed.
Private or unpublished repos appear only when a public launch plan names them.

## Conformance claims

Conformance claims MUST link a published conformance report.
Without a published conformance report, an entry MUST NOT claim compatibility.
Use factual status words only when the linked project states that status.

## Never do this

- Never rank entries.
- Never add marketing adjectives.
- Never claim compatibility without a conformance report.
- Never prefer projects by ownership, license model, or commercial status.
