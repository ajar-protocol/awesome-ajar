# Contributing to awesome-ajar

Start with the org-wide [CONTRIBUTING.md](https://github.com/ajar-protocol/.github/blob/main/CONTRIBUTING.md), org [AGENTS.md](https://github.com/ajar-protocol/.github/blob/main/AGENTS.md), and this repo's `AGENTS.md`, which is the binding directory policy.

New contributors should follow the org-wide [ONBOARDING.md](https://github.com/ajar-protocol/.github/blob/main/ONBOARDING.md) before opening directory PRs.

This repo is a PR-driven directory for Ajar implementations, integrations, tools, bridges, and conformant sites. Entries are welcome any time.

## Wanted now

- Public implementations, tools, bridges, and site entries with working links.
- Corrections to descriptions, licenses, conformance status, and repo ownership.
- Categories that help implementers find real software without ranking projects.
- Links to benchmark reports and demos as they publish.

## Gated work

- "Ajar Compatible" claims require a conformance report once the public harness lands.
- Private repos should be named only when the launch plan already names them.
- Badges or rankings wait for explicit governance criteria.

## How work is tracked

Open issues in this repo. Tasks are aggregated on the org Project board, [Ajar Roadmap](https://github.com/orgs/ajar-protocol/projects).

Use task IDs in titles when one exists. One directory update is one PR unless a broader cleanup task says otherwise.

The DoD is binding. For directory entries, the evidence is a working link, one-line description, license or terms when applicable, and any claimed conformance report.

## Listing rules

- No pay-to-play placement.
- No ranking editorializing.
- Closed-source and commercial entries use the same listing criteria as open-source entries.
- Remove dead links when they cannot be corrected.
